# v0.2.0 Roadmap

Theme: Productized Self-Hosted Demo.

The release goal is to make the app feel like a real product demo while remaining self-hostable and free to run locally. It should not require paid hosting, a paid domain, or project-owner API spend.

## Completed Or In Progress

- Demo Mode: guarded local demo seed endpoint and dashboard CTA.
- First-run onboarding: dashboard setup checklist based on repositories, settings, and review history.
- Local webhook workflow: Cloudflare Tunnel/ngrok docs and repository page guidance.
- Ollama-first/BYOK settings: local Ollama prominence, provider status, encrypted-key and cost responsibility copy.
- Review detail polish: an “About This Review” explainer panel for agent roles.

## Remaining Before Release

- Merge v0.2.0 PRs after required review and green CI.
- Run release verification locally:
  - `docker compose up -d postgres`
  - `python scripts/migrate.py`
  - backend `/health`
  - `pytest -m "not integration" --tb=short -q --cov=backend --cov-report=term --cov-fail-under=60`
  - `pytest -m integration --tb=short -q`
  - `ruff check backend`
  - `npm test -- --run --coverage`
  - `npm run build`
  - `npm audit`
  - `python -m pip_audit -r requirements.txt`
- Confirm README and demo docs match the final merged flow.
- Trigger the release workflow with `v0.2.0`.

## Explicit Non-Goals For v0.2.0

- Paid hosting requirement.
- Paid domain requirement.
- Project-owner managed Claude/OpenAI spend.
- Full SaaS billing or multi-tenant operations.
- Major GitHub App workflow redesign. That belongs in v0.3.0.
